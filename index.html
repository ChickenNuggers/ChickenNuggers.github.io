FusionScript

FusionScript is a C-like language that (at the moment) compiles to Lua code; the project also includes a considerably large set of modules included to the additional standard library. The main inspiration from the language is Python's generators - a large intended use of FusionScript is the asynchronous functions which can be used with the `iterable` library. Overall the language works best with minimal callbacks, based on coroutines and data streams. Some related repositories are listed below.

  • FusionScript
    • ·
  • VIM Editor Support
    • ·
  • Boilerplate Project

I've worked on FusionScript for several months but only during December 2016 have I been able to get a working parser; previous attempts included a failed LPeg lexer and a non-working C++ tokenizer. The version of the parser as of December 2016 is implemented using LPeg. The language itself ships with some additional modules; a class library, a library for functional routines, and a library for iterators. They each are built as an extension to the language and need to be required to be used when using the stock Lua VM.

The language is open to changes; however, because of the current version, code written in FusionScript is likely to change from the current version to FusionScript 1.0.0. Each change that is not compatible with the previous version, until version 1.0+, will be represented by an increase in the subversion. This will also include a list of backwards incompatible changes with the commit logs and release messages that describe how to deal with the changes.

Hashbang

Hashbang, Inc. is an IRC-based international educational network for hackers, developers, and Linux users. I personally volunteer as a system administrator and I also help review changes to the inner infrastructure. I've been with Hashbang, Inc. since May 2015 and have since contributed largely to both the repositories and unrecorded administration of the servers. Below are some related repositories that I have contributed to:

  • Hashbang Website
    • ·
  • User Dotfiles
    • ·
  • Server Configuration

Currently, registration can be done in any POSIX compatible environment with ssh, curl, or the busybox utility set. It's also recommended to have gpg installed for verification of the registration shell script. To register, run the command: sh <(curl https://hashbang.sh | gpg). After registration, you will be put in a tmux session running weechat (an IRC client), Mutt (a mail client), and (by default) a bash shell. From there, feel free to do (almost) whatever.

The entire setup is almost entirely reproducible and submitting a pull request or adding issues is a great way to contribute. Almost any repository in the Hashbang GitHub organization can be modified to change items on the server. Additionally, after enough support, a user can become an administrator themselves to help further improve the #! infrastructure.
